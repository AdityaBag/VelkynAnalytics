Quick Start

  1. Start backend service and frontend static host.
  2. Open platform in browser and verify Dashboard loads.
  3. Run an engine page first, then review consolidated Dashboard.
  4. Use Results page for artifact browsing and History for run audit.

Environment

  • Frontend: localhost:5500
  • Backend: localhost:8000
  • API Docs: backend /docs route
  • Persistence: results/data/history_runs.jsonl + session folders

Settings Policy

  • Settings page is currently showcase mode for v2 UX signaling.
  • Provider upgrade controls are intentionally locked for public users.
  • Preview save/reset actions do not alter live engine behavior.
  • Full profile-based settings persistence is planned in v2.

Troubleshooting

  • Port Busy: Stop existing listeners on 5500/8000 before restart.
  • Stale UI: Use cache-busting query suffix after frontend updates.
  • Blank Chart: Confirm backend endpoint returns expected payload rows.
  • No History Rows: Verify filters are set to intended period (default = current week).
  • Clear History Denied: Operation is restricted to local-machine requests.
  • Live Feed Delay: Use manual refresh and check backend response status.
  • Table Not Updating: Confirm module script cache version is latest.
  • General Recovery: Perform clean restart of backend + frontend services.

Roadmap (v2)

  • User accounts and permissioned profiles
  • Provider upgrades and key-based API routing
  • Cloud backend deployment mode and remote endpoint management
  • Persistent user settings and environment-level policies
  • Expanded report agent automation and deployment workflows
About Velkyn Analytics

Velkyn Analytics Platform

Velkyn Analytics is a modular quantitative research platform built for option valuation, risk diagnostics, and decision-grade scenario workflows with a live market context.

Founder

Aditya Bag is a quantitative developer and engineer based in London, UK, with an engineering background spanning Motorsport & Automotive Engineering (MSc, Brunel University London) and Mechanical Engineering – Automobile (B.Tech, KIIT University).

The platform combines mathematical modelling, practical software architecture, and frontend usability to bridge research experimentation and production-style analytics.

Velkyn Analytics began as an R&D initiative and evolved into a cohesive stack of pricing engines, comparison dashboards, results management, and live-data modules focused on reproducibility, numerical integrity, and scalability.

Platform Design

  • Unified Workflow: run focused engines independently, then inspect a consolidated dashboard view.
  • Mathematical Core: Black-Scholes analytics, Monte Carlo simulation, binomial trees, and IV structures.
  • Risk + Sensitivity: Greeks, scenario shocks, multi-rate analysis, and convergence diagnostics.
  • Operational Layer: batch execution, historical run logs, and organized results sessions.
  • Live Context: market snapshots and rolling series to ground model runs in current conditions.

Engine Methods & Outputs

Monte Carlo Engine

Inputs: $S_0, K, r, \sigma, T,$ paths $M,$ steps $n,$ option type.

Method: GBM dynamics $dS_t = \mu S_tdt + \sigma S_tdW_t$, simulated in discrete time, with variance-reduction methods (antithetic, control variate, moment matching, variable-day horizon).

Outputs: option price, standard error, confidence interval, path chart, terminal distribution.

Black-Scholes Engine

Inputs: $S, K, r, \sigma, T,$ option type, market option price (for IV).

Method: closed-form pricing with $d_1 = \frac{\ln(S/K) + (r + \frac{1}{2}\sigma^2)T}{\sigma\sqrt{T}}$, $d_2 = d_1 - \sigma\sqrt{T}$ and normal CDF $N(\cdot)$.

Outputs: call/put values, Greeks (Delta, Gamma, Vega, Theta, Rho), implied volatility solution.

Binomial Tree Engine

Inputs: $S, K, r, \sigma, T,$ steps $N,$ option type/style.

Method: CRR tree with $u=e^{\sigma\sqrt{\Delta t}}$, $d=1/u$, risk-neutral probability $p=\frac{e^{r\Delta t}-d}{u-d}$ and backward induction.

Outputs: European and American prices, BS benchmark comparison, convergence behavior.

Volatility Engine

Inputs: base volatility, strike/maturity ranges, rate and spot context.

Method: synthetic smile/surface construction over strike-tenor grids to visualize implied-volatility structure and skew/term behavior.

Outputs: smile curve chart and surface snapshot for relative vol structure analysis.

Multi-Rate Engine

Inputs: ticker, $K, T, \sigma,$ option type, list of rates $\{r_i\}$.

Method: reprice the same contract under each rate scenario, i.e. evaluate $C(r_i)$ or $P(r_i)$ across the grid.

Outputs: scenario count, baseline spot, and rate-vs-price sensitivity curve.

Scenario Engine

Inputs: baseline option parameters plus spot/volatility shock vectors.

Method: stress pricing under transformed states $S' = S(1+\delta_S)$, $\sigma' = \sigma(1+\delta_\sigma)$ and recompute option value per scenario.

Outputs: scenario table and comparative chart of stressed option prices.

Batch & History Modules

Inputs: multi-ticker set and shared model parameters; run metadata over time.

Method: vectorized/repeated pricing execution with structured logging and retrieval.

Outputs: consolidated batch result table and persistent run-audit history.

Live Market & Dashboard

Inputs: selected asset class/symbol, refresh cadence, display mode.

Method: stream quote/series data, derive OHLC and volume context, aggregate cross-engine summaries on dashboard.

Outputs: live pricing panels, gradient/candlestick charts, volume overlay, and real-time summary views.
